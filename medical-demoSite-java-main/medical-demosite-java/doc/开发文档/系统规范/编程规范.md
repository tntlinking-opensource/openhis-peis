## 二、对象注入规范

**1. 依赖注入**

- 实现依赖或者对象注入时统一规范使用spring 官方推荐的构造注入`@RequiredArgsConstructor`，构造器注入能够保证注入的组件不可变，并且确保需要的依赖不为空。此外，构造器注入的依赖总是能保证完全初始化的状态。

```java

@Slf4j
@Service
@RequiredArgsConstructor
public class UserService extends ServiceImpl<UserDao, User> {

    private final CompanyService companyService;
    private final RoleService roleService;
    private final SmsService smsService;
    private final CacheService cacheService;
}
```

- 如果需要注入的类有多个实现，需要用 @Qualifier 注解指定要注入的 bean 实现，如 OrderService 有两个实现 bean，一个 bean name 是 "orderServiceV1"，另一个是 "
  orderServiceV2"，假设要注入 "orderServiceV2"

```java

@Service
@RequiredArgsConstructor
public class UserServiceImpl implements UserServiceService {

    @Qualifier("orderServiceV2")
    private final OrderService orderService;
    private final AdminService adminService;

}
```

**2. 属性注入**

- 外部属性注入统一规范使用`@Value`，如注入yml或者properties文件中的属性
- 3种使用方式

```java
/**1.普通注入**/
@Value("jdbc:mysql://localhost:3306/test")
private String url;
@Value("com.mysql.jdbc.Driver")
private String driveName;
@Value("Scott")
private String userName;
@Value("10")
private int no;

/**2.注入配置文件 `@Value("${}")`**/
@Value("${mysql.pwd}")
private String password;

/**2.注入表达式并运算：可以使用SqEL表达式**/
@Value("#{T(java.lang.Runtime).getRuntime().availableProcessors() * 2}")
private int maxCons;
```
