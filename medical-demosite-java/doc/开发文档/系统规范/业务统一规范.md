## 业务统一规范

### 1.数据表记录的id(String类型的主键ID)生成规范

统一使用雪花算法生成纯数字ID

```java

private final Snowflake snowflake;

        String id=String.valueOf(snowflake.nextId());

```

### 2.用户ID使用规范

为了适配原体检系统的数据，在新系统中的管理员表里新增一个字段`用户编号：user_no`， 该字段对应的是原系统的`user_id`，所以在所有已`md_`为前缀的数据表中设计到`userId`的， 统一使用`userNo`

```java
String userNo=SecurityUtils.getUserNo();

        createCombo.setModifier(SecurityUtils.getUserNo());
```

### 3.部门ID(科室ID)

为了适配原体检系统的数据，在新系统中的部门表里新增一个字段`部门编号：dept_no`， 该字段对应的是原系统的`dept_id`，所以在所有已`md_`为前缀的数据表中设计到`deptId`的，如部门id、科室id等，
统一使用`deptNo`，选择科室时，携带给后端的参数科室id使用`deptNo`

```java
SysDept dept=sysDeptMapper.getByDeptNo(items.getIdDepart());
```
